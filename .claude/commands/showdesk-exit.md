# 세션 종료 — 작업 기록 자동화

세션에서 완료된 작업을 DEV_LOG.md와 작업실록.md에 기록하고 GitHub에 푸시한다.

## 실행 순서

### 1. 이번 세션 작업 파악

```bash
# 이번 세션 커밋 목록
git -C /c/dev/ShowDesk log --oneline -20

# 현재 미커밋 변경사항
git -C /c/dev/ShowDesk status
git -C /c/dev/ShowDesk diff
```

대화 히스토리도 참고해 커밋에 포함되지 않은 작업(디버깅, 조사, 의사결정)도 놓치지 않는다.

### 2. 미커밋 변경사항 처리

코드 변경사항이 있으면 적절한 커밋 메시지로 먼저 커밋한다.
문서(DEV_LOG.md, 작업실록.md, DECISIONS.md) 변경은 마지막에 별도로 커밋한다.

커밋 컨벤션:
- `feat:` 새 기능
- `fix:` 버그 수정
- `docs:` 문서
- `chore:` 설정/의존성
- `design:` UI/디자인 변경
- `3d:` 3D 렌더러/뷰어 변경
- `bom:` BOM 파싱/매핑 변경

### 3. DEV_LOG.md — 현재 세션 컨텍스트 갱신

`DEV_LOG.md` 상단 "현재 세션 컨텍스트" 표를 오늘 날짜로 갱신한다.

업데이트 항목:
- 날짜 (최종 갱신)
- 브랜치 및 최신 커밋 해시
- 미커밋 변경 여부
- 완료된 기능 목록 (신규 완료분 추가)
- 다음 추천 작업 (완료 항목 제거, 새 항목 추가)
- 알려진 이슈 (해결된 항목 제거, 신규 추가)

### 4. 작업실록.md — 오늘 날짜 항목 추가

`작업실록.md`에 오늘 날짜(`## YYYY-MM-DD`) 항목을 추가한다.

- 백로그 섹션 바로 위에 오늘 날짜 항목 삽입
- 오늘 날짜 항목이 이미 있으면 내용 이어서 추가
- 완료된 기능은 백로그 목록에서 제거

작성 기준:
- 구현한 기능: 핵심 파일, 변경 내용 위주로 간결하게
- 수정한 버그: 원인과 해결 방법을 한 줄로
- 의사결정: 왜 그렇게 했는지 이유 포함
- DECISIONS.md의 미결 사항(D-*) 중 확정된 항목 업데이트
- 미완료 항목: 다음 세션 인계 내용 명시

### 5. DECISIONS.md 갱신

이번 세션에서 결정된 사항이 있으면:
- 미결 항목(D-*)을 확정 항목으로 이동
- 새로운 미결 사항 추가

### 6. ROADMAP.md 갱신

완료된 체크박스를 체크한다.

### 7. 문서 커밋 및 푸시

```bash
git -C /c/dev/ShowDesk add DEV_LOG.md 작업실록.md DECISIONS.md ROADMAP.md
git -C /c/dev/ShowDesk commit -m "docs: 세션 종료 — 작업 기록 갱신 ($(date +%Y-%m-%d))"
git -C /c/dev/ShowDesk push
```

### 8. 완료 보고

```
[ShowDesk 세션 종료]
완료한 작업: <목록>
업데이트한 문서: <목록>
커밋: <커밋 해시>
push: 완료
다음 할 일: <1~3개>
```

## 주의사항

- 커밋 전 민감한 파일(.env, 키 파일 등) 스테이징 여부 확인
- 충돌이 있으면 push 전에 사용자에게 알리고 수동 해결 요청
