# ShowDesk — 설계 결정 로그

미결 항목은 `D-숫자` 태그로 표기한다.

---

## 확정된 결정

### 렌더링 엔진
- **결정**: Three.js (react-three-fiber 없이 직접 사용)
- **이유**: STLLoader 기본 내장, 커뮤니티 풍부, Electron 환경과 호환성 검증됨
- **대안**: Babylon.js (PBR 품질 높으나 학습 비용), react-three-fiber (추상화 레이어 불필요한 복잡성 추가)

### 앱 프레임워크
- **결정**: Electron + electron-vite + React
- **이유**: Windows PC 데스크탑 앱, 로컬 파일 접근 필수, 오프라인 동작 요건
- **대안**: Tauri (Rust 기반, 빠르나 생태계 작음)

### 상태 관리
- **결정**: Zustand
- **이유**: 보일러플레이트 최소, Three.js 씬 상태와 React UI 상태 분리 용이
- **대안**: Redux (과함), Jotai (원자 모델이 3D 상태에 부적합)

### 지원 파일 포맷 (MVP)
- **결정**: STL만 지원 (ASCII + Binary)
- **이유**: MVP 범위 최소화, 가장 보편적인 3D 출력/공유 포맷
- **향후**: STEP/IGS는 Phase 2 이후 별도 변환 파이프라인으로 추가

### BOM 매핑 키
- **결정**: STL 파일명(확장자 제외) = BOM의 `part_number`
- **이유**: 가장 단순한 매핑 방법, 별도 메타데이터 불필요
- **주의**: 파일명 네이밍 컨벤션을 사용자에게 명확히 안내 필요

## 미결 사항

### D-1: ✅ 해결됨 — BOM-first + 파트별 STL 수동 지정 방식 채택
- **결정**: BOM을 먼저 로드한 후 파트 트리에서 각 파트에 STL 파일을 개별 지정
- **이유**: 어셈블된 단일 STL은 파트 경계 정보가 없어 자동 분리 불가. 파일명 컨벤션 강제 없이 유연하게 매핑 가능
- **STL-only 모드도 유지**: 단일 파일 빠른 뷰 진입점 별도 제공
- **프로젝트 파일(.showdesk)**: BOM 경로 + 파트별 STL 경로 JSON으로 저장, 재오픈 시 전체 복원

### D-2: 파트 하이라이트 색상 커스터마이징
- **질문**: 하이라이트 색상을 사용자 설정으로 변경 가능하게 할지
- **현재 방향**: MVP는 고정 색상(주황 #FF6B00), Phase 2에서 설정 추가 검토
- **결정 필요 시점**: Phase 1 완료 후 사용자 피드백 반영
